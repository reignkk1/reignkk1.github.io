렌더링 엔진 동작과정

렌더링 엔진은 URI를 통해 요청을 받아서 해당하는 데이터를 렌더링하는 역할을 담당합니다. chrome 과 IOS 는 webkit이라는 렌더링 엔진을 사용합니다.

대략적인 렌더링 엔진 동작 과정

  1. DOM tree 구축을 위한 HTML Parsing, CSS, JavaScript Parsing => HTML 문서를 파싱한 후 , Content tree 내부에서 Tag(a,div)를 DOM Node로 변환합니다. 그 다음 CSS파일과 함께 모든 스타일 요소를 파싱합니다. 스타일 요소와 HTML 표시 규칙, JavaScript의 파싱 결과물은 render tree를 생성합니다.

  2. Render tree 구축: HTML과 CSS를 파싱해서 만들어진 Render tree는 정해진 순서대로 렌더링한다.

  3. Render tree 배치: render tree가 생성이 끝나면, 배치가 시작된다. 각 Node가 정확한 위치에 표시되기 위해 이동합니다.

  4. Render tree 그리기: 각 Node를 배치 완료하면 UI에서 paint 작업을 합니다.

1번과 2,3,4번은 병렬적으로 처리됩니다.

서버와 통신하여 Data를 계속 받아오면서 받아온 HTML, CSS, JavaScript를 파싱한 후 (1번) Render tree에 Node를 그립니다.(2,3,4번)

webkit 의 동작 과정

1. HTML을 Parsing 하여 DOM tree를 생성한다

아래의 HTML을 파싱하여 DOM tree를 생성한다. (DOM으로 바꿔야 JavaScript가 조작할 수 있다.

DOM 참조 - https://developer.mozilla.org/ko/docs/Web/API/Document_Object_Model/Introduction

<html> <body> <h1>Hello, World</h1> <div> <span>Naver</span> </div> <script></script> </body> </html>

브라우저는 태그의 파싱과 실행을 동시에 진행합니다. 그러므로 아래 과정으로 HTML 태그를 파싱합니다.

  1. script 태그를 파싱한다.
  2. script 태그를 실행한다.
  3. 실행이 완료된 후 다음 태그를 파싱한다.

위 과정에서 HTML 파싱을 멈추게 된다는 특징이 있으므로 script 태그는 body태그의 밑에 두는 것이 좋다.

2. CSS(style sheet)를 파싱하여 스타일 규칙을 얻는다.

CSS를 파싱하여 CSSOM생성한다

3. DOM tree를 생성하는 동시에, 이미 생성된 DOM tree에 스타일 규칙을 적용한다.

  1. DOM tree를 구성하는 하나의 DOM node는 attach라는 method를 가진다. 새로운 DOM node가 추가되면 attach가 호출되어 render object를 생성한다.

  2. Render Object는 Render tree의 구성요소로서 자신과 자식요소를 어떻게 배치하고 그려야할지 안다.

  3. node의 css box를 표시할 정보를 갖고 있음.

  4. 모든 DOM node가 전부 Render Object로 생성되는 것은 아님. (head 태그, display:none 등)

4. 구축한 Render tree를 배치(Layout)한다.

배치는 html 태그 요소에 해당하는 최상위 Render Object에서 시작한다. 화면에 왼쪽 위 부터 Render Object에 해당하는 DOM node를 그려나간다.

5. 배치가 끝난 Render tree를 그린다.

Render Tree 탐색 후 해당하는 Render Object의 paint method를 호출한다.
