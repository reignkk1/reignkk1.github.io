캐싱은 왜 사용하는 것인가?

캐싱

어떤 데이터를 한번 받아온 후에 그 데이터를 불러온 저장소 보다 가까운 곳에 임시로 저장하여, 필요시 더 빠르게 불러와서 사용하는 프로세스를 의미한다.

장점

  • 캐시에 데이터를 미리 복사해 놓으면 데이터를 다시 가져오지 않고 빠르게 데이터에 접근할 수 있다.
  • 반복적으로 동일한 결과를 가져와야할 때 접근 시간을 절약할 수 있다.
  • 다시 요청해서 데이터를 가져오지 않기에 서버 부하를 줄일 수 있다.

단점

  • 캐시에 저장할 수 있는 데이터의 양에는 제한이 있기 때문에, 메모리 용량 부족으로 성능 저하가 발생할 수 있다.
  • 캐시에 저장된 데이터가 실제 데이터와 일치하지 않을 경우 문제가 발생할 수 있다.

브라우저 캐싱(Browser Caches)

브라우저는 HTML,CSS,JS,Img 파일 등의 정적 리소스를 캐시한다. 따라서 이전에 방문했던 페이지의 리소스들을 서버에 다시 요청하지 않고 바로 캐싱된 리소스를 응답 받아 웹 페이지가 빠르게 느껴지고 서버는 불필요한 요청을 받지않아 부담이 줄어든다.

프록시 캐싱(Proxy Caches)

웹 브라우저와 서버 사이에 proxy라는 것을 두는 방식이다. 중간에서 요청을 조작하여 대신 해주는 것을 ‘프록시’라고 한다. 프록시의 경우 여러 컴퓨터로부터 요청을 받고 처리하다 보니 같은 데이터가 자주 접근될 확률이 높아져서 캐시의 성능이 좋아진다. 조직 내에서 접속하는 웹 사이트의 리소스들을 캐시하여 속도가 빨라지고 네트워크 사용량을 줄임

메모이제이션

메모이제이션은 함수의 결과를 캐싱하여 다음에 같은 함수를 호출할 때 다시 계산할 필요 없이 이전에 계산한 값을 반환하는 기술이다.
